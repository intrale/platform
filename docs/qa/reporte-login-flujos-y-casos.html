4. Flujo A — Login Basico (email + password)

4.1 Descripcion funcional PO

El usuario ingresa su email y contrasena en la pantalla de Login. El sistema valida los campos con Konform en frontend, envia POST a /{biz}/signin, Cognito autentica, el backend verifica que el usuario tenga perfil APPROVED en el business, y retorna tokens JWT. El token se almacena en storage local para auto-login futuro.

Precondiciones: App instalada | Conexion a internet | Usuario registrado y confirmado en Cognito | Perfil con state=APPROVED en DynamoDB

5. Flujo B — Auto-Login (sesion anterior)

5.1 Descripcion funcional PO

Al abrir la app, un LaunchedEffect en Login.kt verifica si hay un token guardado en CommKeyValueStorage. Si existe, ejecuta previousLogin() y navega automaticamente al Home sin que el usuario deba ingresar credenciales.

Precondiciones: Login exitoso previo | Token almacenado en KeyValueStorage | App cerrada y reabierta

5.2 Diagrama de flujo

LaunchedEffect(Unit)"]
  LOGIN --> CHECK{"previousLogin()
token != null?"}
  CHECK -->|"Token existe"| TRIGGER["triggerLogin()
auto-login"]
  CHECK -->|"Sin token"| WAIT["Mostrar formulario
esperar input"]
  TRIGGER --> VALIDATE["Verificar token
con backend"]
  VALIDATE -->|"Token valido"| HOME["Navegar a Home
(sin mostrar login)"]
  VALIDATE -->|"Token expirado/invalido"| CLEAR["Limpiar token
mostrar formulario"]
  CLEAR --> WAIT

6. Flujo C — Cambio de Contrasena Forzado

6.1 Descripcion funcional PO

Cuando Cognito retorna el challenge NEW_PASSWORD_REQUIRED (primer login del usuario con password temporal), la app muestra campos adicionales: nueva contrasena, nombre y apellido. El backend llama adminRespondToAuthChallenge, actualiza atributos del usuario y re-autentica con la nueva contrasena.

Precondiciones: Usuario recien creado en Cognito con password temporal | Primer intento de login | Cognito fuerza cambio de contrasena

7. Flujo D — Logout

7.1 Descripcion funcional PO

El usuario cierra sesion desde el Home. El sistema ejecuta DoResetLoginCache que limpia token, cache de perfil e idioma preferido, luego SessionStore.clear() reinicia el estado de sesion, y finalmente navega de vuelta a la pantalla de Login.

Precondiciones: Sesion activa | Token almacenado | Usuario en Home o pantalla autenticada

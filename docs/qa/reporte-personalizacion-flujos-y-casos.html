1. Resumen Ejecutivo

La funcionalidad Personalización del Negocio permite a los administradores de un negocio configurar la apariencia visual de su tienda en la plataforma Intrale. Esta pantalla está ubicada en /personalization y es accesible únicamente para roles BusinessAdmin y PlatformAdmin con un selectedBusinessId activo.


Estado al 2026-03-12: La pantalla existe como placeholder parcial. De las 5 secciones previstas, solo Tipografía tiene implementación real completa. Las secciones Colores, Imágenes, Ícono de App y Preview son placeholders con label "pendiente". Este reporte sirve como especificación funcional para cuando se implementen las secciones restantes.

Recomendaciones de implementación

  1. Orden sugerido: Colores → Preview → Imágenes → Ícono (de menor a mayor complejidad)
  2. Patrón a seguir: Usar TypographyScreen como modelo de implementación para Colores y Preview
  3. Backend: Implementar SecuredFunction para cada sección con validación BusinessAdmin || PlatformAdmin
  4. Tests: Agregar tests Maestro E2E para PersonalizationScreen (A-01 a A-07) como primer paso
  5. Imágenes: Requiere definir política de almacenamiento S3 y CDN antes de implementar
  6. Ícono de App: Requiere definir el mecanismo de distribución (¿OTA update? ¿rebuild del APK?)

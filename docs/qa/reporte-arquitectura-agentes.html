1. Resumen Ejecutivo

Intrale Platform implementa un sistema de agentes IA autonomos orquestados que automatiza el ciclo completo de desarrollo de software: planificacion, implementacion, testing, QA, code review y delivery. Los agentes operan en worktrees Git aislados, se comunican via Telegram y son monitoreados en tiempo real.

6.1 Detalle de Cada Hook

branch-guard.js

PreToolUse[Bash]

Funcion: Intercepta comandos git push. Si la rama actual es main o master, BLOQUEA la ejecucion y devuelve error. Solo permite push a ramas agent/*, codex/*, feature/*, etc.

permission-approver.js

PermissionRequest

Funcion: Cuando Claude necesita aprobacion para una herramienta, envia mensaje a Telegram con botones inline (Permitir / Rechazar). Implementa retry con escalada de urgencia: 4 intentos en ~65 minutos. Si no hay respuesta, cae a prompt local en consola.

post-git-push.js

PostToolUse[Bash]

Funcion: Detecta git push exitoso. Extrae SHA y rama, luego lanza ci-monitor-bg.js en background (proceso hijo desacoplado). Este daemon pollea GitHub Actions cada 30s durante max 20 minutos y envia resultado a Telegram.

activity-logger.js

PostToolUse[*]

Funcion: Registra cada herramienta ejecutada (excepto lecturas) en activity-log.jsonl. Actualiza la sesion en sessions/*.json con conteo de acciones, ultima herramienta, progreso de tasks (sub-pasos). Arranca el reporter PNG automatico si no esta corriendo.

permission-tracker.js

PostToolUse[*]

Funcion: Auto-aprende patrones de permisos exitosos. Si una herramienta se ejecuta sin rechazo, persiste el patron en settings.local.json (allow). Esto evita re-preguntar por la misma accion en futuras sesiones. Respeta deny[] explicitos.

post-issue-close.js

PostToolUse[Bash]

Funcion: Detecta gh issue close. Via GraphQL, mueve el issue a la columna "Done" en el Project V2 de GitHub. Automatiza la actualizacion del board sin intervencion manual.

notify-telegram.js

Notification

Funcion: Reenvia cualquier notificacion generica de Claude Code a Telegram con formato HTML enriquecido. Incluye contexto del agente activo, rama y tarea actual. Ignora permission_prompts (los maneja permission-approver).

stop-notify.js

Stop

Funcion: Al terminar Claude, marca la sesion como "done" en su JSON, limpia sesiones antiguas (>2h), y envia un resumen final a Telegram (intenta como imagen PNG, fallback a texto HTML).

11.3 Convenciones Criticas

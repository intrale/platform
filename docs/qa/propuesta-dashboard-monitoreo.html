1 Vision: un solo dashboard, tres canales Guru + UX
Cambio de paradigma El dashboard deja de ser un script de terminal con caracteres ASCII. Pasa a ser una pagina web autosuficiente (un solo archivo HTML con datos embebidos) que se actualiza en tiempo real y se consume desde tres canales: navegador local, Telegram (como screenshot PNG) y consola de Claude Code (resumen textual).
🌐 Navegador (local)
Pagina web servida por dashboard-server.js en localhost:3100. Auto-refresh via SSE (Server-Sent Events) cada 5 segundos. Graficos reales, animaciones CSS, hover tooltips. Siempre abierta en una pestana del navegador.
📱 Telegram (remoto)
Puppeteer toma screenshot del dashboard web a 375px de ancho (mobile viewport). Envia la imagen con botones inline para acciones rapidas. Diferenciar heartbeat (silencioso) vs alerta critica (con vibracion).
💻 /monitor (consola)
El agente /monitor genera un resumen textual compacto para Claude Code. No intenta replicar el dashboard visual — solo muestra KPIs, alertas criticas y estado de agentes. Util cuando no hay navegador disponible.

4 Arquitectura tecnica Guru

El nuevo dashboard reemplaza el script monolitico dashboard.js con un servidor web liviano que sirve un HTML con datos actualizados via SSE. Puppeteer lo screenshotea para Telegram.

flowchart TB subgraph Agentes["Sesiones Claude Code"] A1["Agente 1"] --> H A2["Agente 2"] --> H A3["Agente N"] --> H end H["activity-logger.js\n(PostToolUse hook)"] H --> S1["sessions/*.json"] H --> S2["activity-log.jsonl"] subgraph Server["dashboard-server.js (Node)"] WS["HTTP Server\nlocalhost:3100"] SSE["SSE endpoint\n/events"] SC["Screenshot endpoint\n/screenshot"] end S1 --> WS S2 --> WS WS --> SSE WS --> SC SSE --> BR["🌐 Navegador\n(auto-refresh 5s)"] SC --> PUP["Puppeteer\n375px viewport"] PUP --> TG["📱 Telegram\n(foto + inline buttons)"] WS --> CLI["💻 /monitor\n(resumen JSON)"]

ComponenteResponsabilidadEndpoint

GET /api/statusJSON con KPIs para que /monitor consuma sin parsear HTMLlocalhost:3100/api/status

Auto-inicio

// activity-logger.js detecta actividad de agente y arranca el servidor
if (!serverRunning) {
  spawn('node', ['dashboard-server.js'], { detached: true, stdio: 'ignore' });
}
// Auto-stop: si no hay sesiones activas por 30 minutos, el servidor se cierra solo

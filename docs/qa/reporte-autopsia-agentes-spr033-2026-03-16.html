1. Hallazgo principal: Downgrade silencioso de modelo

CRITICO: Los 3 agentes arrancaron usando Sonnet 4.6 pero Claude Code los fue degradando automaticamente a Haiku 4.5 durante la sesion. Esto ocurrio porque Run-AgentStream.ps1 no pasaba el flag --model al CLI, permitiendo que el sistema eligiera el modelo segun disponibilidad/carga.

4. Por que murio el watcher?

El watcher no crasheo. Su ultima linea de log es un estado normal: Estado: activos=0/3 · cola=0 · slots libres=3.

Murio porque:

  • Fue lanzado como proceso hijo de Start-Agente.ps1 con Start-Process -WindowStyle Hidden
  • Cuando la sesion PowerShell del usuario se cerro (o Windows termino la sesion), los procesos hijos fueron terminados
  • El watcher no tiene mecanismo de auto-restart ni servicio Windows

No es un bug — es el comportamiento esperado. El watcher deberia auto-terminarse cuando no hay trabajo, lo cual no hacia (seguia polleando indefinidamente con 0 activos).

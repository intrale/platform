4. Flujo 1 — Listar Productos con Filtros

4.1 Descripcion funcional PO

BusinessAdmin o PlatformAdmin acceden a la lista de productos del negocio. La lista muestra nombre, precio, unidad, categoria, estado (Draft/Published) y disponibilidad. Se puede filtrar por categoria usando FilterChips.

Precondiciones: Login como BusinessAdmin o PlatformAdmin | Negocio seleccionado (APPROVED) | Conexion a internet

4.2 Diagrama de flujo

flowchart TD
  START(("Dashboard")) --> NAV["Navega a<br>/business/products"]
  NAV --> LOAD{"Negocio<br>seleccionado?"}
  LOAD -->|"No"| MISSING["Estado:<br>MissingBusiness<br>Mensaje de error"]
  LOAD -->|"Si"| FETCH["GET /business/{bizId}/products<br>+ GET categories"]
  FETCH --> RES{"Resultado?"}
  RES -->|"Error de red"| ERR["Estado: Error<br>Snackbar + Retry"]
  RES -->|"Lista vacia"| EMPTY["Estado: Empty<br>Mensaje 'Sin productos'"]
  RES -->|"Productos OK"| LOADED["Estado: Loaded<br>LazyColumn con ProductCards"]
  LOADED --> FILTER["FilterChips por categoria<br>'Todas' | cat1 | cat2 | ..."]
  FILTER -->|"Selecciona categoria"| FILTERED["Filtra items localmente<br>por categoryId"]
  FILTERED --> LOADED
  LOADED -->|"Tap en producto"| DRAFT["ProductEditorStore.setDraft()<br>Navega a form (Edit)"]
  LOADED -->|"FAB: Agregar"| NEW["ProductEditorStore.clear()<br>Navega a form (Create)"]
  ERR -->|"Retry"| FETCH

5. Flujo 2 — Crear Producto

5.1 Descripcion funcional PO

BusinessAdmin crea un nuevo producto completando el formulario con nombre, precio, unidad y categoria (obligatorios), mas descripcion, estado, disponibilidad y stock (opcionales). El sistema valida con Konform en frontend y envia POST al backend.

Precondiciones: Login como BusinessAdmin/PlatformAdmin | Negocio seleccionado | Al menos una categoria existente | Desde ProductListScreen toco FAB "Agregar"

5.2 Diagrama de flujo

flowchart TD
  START(("FAB: Agregar")) --> CLEAR["ProductEditorStore.clear()"]
  CLEAR --> NAV["Navega a /business/products/form<br>modo: Create"]

  CATS -->|"Error"| ECAT["Error cargando categorias<br>Retry disponible"]

  FORM --> INPUT["Usuario completa campos"]

6. Flujo 3 — Editar Producto / Draft

6.1 Descripcion funcional PO

BusinessAdmin edita un producto existente. Al hacer tap en un ProductCard, se crea un ProductDraft via ProductEditorStore y se navega al formulario en modo Edit. Los campos se pre-llenan con los datos actuales del producto. Al guardar se hace PUT al backend.

Precondiciones: Login como BusinessAdmin/PlatformAdmin | Producto existente en la lista | Desde ProductListScreen tap en ProductCard

6.2 Diagrama de flujo

flowchart TD
  START(("Tap en ProductCard")) --> DRAFT["ProductListViewModel.toDraft()<br>Convierte ProductListItem a ProductDraft"]
  DRAFT --> STORE["ProductEditorStore.setDraft(draft)"]
  STORE --> NAV["Navega a /business/products/form"]
  NAV --> APPLY["ProductFormViewModel.applyDraft()<br>Detecta id != null → modo Edit"]
  APPLY --> LOAD["ensureProductLoaded()<br>GET /products/{id} si necesario"]
  LOAD --> CATS["Carga categorias"]
  CATS --> FORM["Formulario pre-llenado<br>con datos del draft"]
  FORM --> EDIT["Usuario modifica campos"]
  EDIT --> SAVE["Toca 'Guardar'"]
  SAVE --> VAL{"Validacion<br>Konform"}
  VAL -->|"Invalido"| ERR["Errores inline"]
  VAL -->|"Valido"| PRICE{"Precio > 0?"}
  PRICE -->|"No"| EPRICE["Error precio"]
  PRICE -->|"Si"| HTTP["PUT /business/{bizId}/products/{id}<br>ProductRequest"]
  HTTP -->|"Error"| ENET["Snackbar error"]
  HTTP -->|"200 OK"| OK["Snackbar exito<br>Draft actualizado"]
  OK --> BACK["Navega a ProductListScreen"]

  ERR --> FORM
  EPRICE --> FORM
  ENET --> FORM

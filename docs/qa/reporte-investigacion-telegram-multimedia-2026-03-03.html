3. Imagenes desde Telegram

3.1 Factibilidad: ALTA

Veredicto: Totalmente viable La API de Claude soporta imagenes como content blocks (base64, URL o Files API). Telegram provee las fotos como JPEG, formato nativamente compatible.

3.2 Formatos soportados por Claude API

FormatoMIMELimiteFuente Telegram
JPEGimage/jpeg5 MBmsg.photo[]
PNGimage/png5 MBmsg.document
GIFimage/gif5 MBmsg.document
WebPimage/webp5 MBmsg.sticker

3.3 Flujo tecnico propuesto

1
Detectar imagen — Verificar msg.photo (array de tamanios) o msg.document con MIME de imagen
↓
2
Obtener file_id — Tomar el ultimo elemento de msg.photo[] (mayor resolucion) o msg.document.file_id
↓
3
Descargar archivo — GET /bot{TOKEN}/getFile?file_id=... → obtener file_path → GET /file/bot{TOKEN}/{file_path}
↓
4
Codificar en base64 — Convertir el buffer binario a string base64 y determinar media_type
↓
5
Llamar API Anthropic — POST /v1/messages con content blocks [{type:"image"}, {type:"text"}]
↓
6
Devolver respuesta — Enviar la respuesta de Claude al chat de Telegram

3.4 Limitacion critica: claude -p no acepta imagenes

El CLI claude -p solo acepta texto via stdin Para enviar imagenes a Claude, se debe llamar directamente a la API REST de Anthropic (https://api.anthropic.com/v1/messages) en lugar de usar executeClaude(). Esto requiere tener ANTHROPIC_API_KEY disponible en el entorno del hook.

3.5 Ejemplo de llamada a la API

// Llamada directa a Anthropic API con imagen const body = { model: "claude-sonnet-4-6", max_tokens: 2048, messages: [{ role: "user", content: [ { type: "image", source: { type: "base64", media_type: "image/jpeg", data: imageBase64 // Buffer.toString("base64") } }, { type: "text", text: caption || "Describe esta imagen y responde en espanol." } ] }] };

4. Audio / Voz desde Telegram (Transcripcion) ACTUALIZADO v2

4.1 Factibilidad: ALTA (revisada al alza)

Veredicto: Altamente viable con OpenAI La API de Claude NO soporta audio como input, pero OpenAI ofrece tres modelos de transcripcion con precios desde $0.003/min y calidad excelente en espanol. La factibilidad se eleva de MEDIA a ALTA gracias a los nuevos modelos GPT-4o Transcribe (marzo 2025).

4.2 Formato de audio en Telegram

4.3 Modelos de transcripcion OpenAI (ChatGPT) NUEVO

OpenAI ofrece actualmente tres modelos de Speech-to-Text, todos via el mismo endpoint:

ModeloIDPrecio/minAcepta .oggCalidadRecomendacion
GPT-4o Transcribe gpt-4o-transcribe $0.006 Si Mejor WER — menor tasa de error, mejor manejo de acentos Recomendado (calidad)
GPT-4o Mini Transcribe gpt-4o-mini-transcribe $0.003 Si Buena — 89% menos alucinaciones vs legacy Recomendado (costo)
Whisper (legacy) whisper-1 $0.006 Si Baseline — modelo V2 large Superado
Ventajas clave de los nuevos modelos (marzo 2025+) • gpt-4o-transcribe: menor Word Error Rate (WER), mejor manejo de ruido y acentos hispanohablantes
• gpt-4o-mini-transcribe: 50% mas barato que Whisper, 89% menos alucinaciones (snapshot dic 2025)
• Speaker diarization disponible con variante gpt-4o-transcribe-diarize (mismo precio)
• Todos usan el mismo endpoint: POST /v1/audio/transcriptions

4.6 Ejemplo de transcripcion (actualizado)

// Transcripcion con OpenAI GPT-4o Mini Transcribe (Node.js puro) async function transcribeAudio(audioBuffer, filename) { const boundary = "----TranscribeBoundary" + Date.now().toString(36); let body = ""; body += "--" + boundary + "\r\n"; body += 'Content-Disposition: form-data; name="model"\r\n\r\n'; body += 'gpt-4o-mini-transcribe\r\n'; // ← NUEVO: 50% mas barato que whisper-1 body += "--" + boundary + "\r\n"; body += 'Content-Disposition: form-data; name="language"\r\n\r\nes\r\n'; const pre = Buffer.from( body + "--" + boundary + "\r\n" + 'Content-Disposition: form-data; name="file"; filename="' + filename + '"\r\n' + 'Content-Type: audio/ogg\r\n\r\n' ); const post = Buffer.from("\r\n--" + boundary + "--\r\n"); const payload = Buffer.concat([pre, audioBuffer, post]); // POST a https://api.openai.com/v1/audio/transcriptions // Headers: Authorization: Bearer {OPENAI_API_KEY} // Content-Type: multipart/form-data; boundary={boundary} return transcriptionText; }

5. Respuesta por Voz (Text-to-Speech) NUEVO en v2

5.1 Factibilidad: ALTA

Veredicto: Totalmente viable y natural para Telegram OpenAI ofrece TTS con salida en formato Opus — el mismo codec nativo de Telegram para mensajes de voz. Esto permite que el bot responda con audio ademas de texto, creando una experiencia conversacional completa (voz-in → voz-out).

5.2 Modelos de Text-to-Speech disponibles

ModeloIDPrecioVocesCalidadFeature clave

TTS HD tts-1-hd $30/1M chars 6 voces Alta Mayor fidelidad
GPT-4o Mini TTS gpt-4o-mini-tts $12/1M output tokens 13 voces Estudio (48kHz) Dirigible — se puede controlar tono, emocion, ritmo

5.4 Dirigibilidad (feature unico de gpt-4o-mini-tts)

Control fino del estilo de habla A diferencia de los modelos legacy, gpt-4o-mini-tts acepta instrucciones de estilo: "Habla en tono amigable y profesional, con ritmo moderado, como un asistente tecnico." Esto permite adaptar la personalidad del bot sin cambiar de voz.

5.5 Flujo tecnico: respuesta por voz

1
Claude genera respuesta — Texto normal del flujo existente
↓
2
Sintetizar voz — POST /v1/audio/speech con model: "gpt-4o-mini-tts", response_format: "opus"
↓
3
Enviar como voice message — sendVoice de Telegram Bot API con el buffer Opus

5.6 Ejemplo de sintesis de voz

// Text-to-Speech con GPT-4o Mini TTS (Node.js puro) async function textToSpeech(text) { const body = JSON.stringify({ model: "gpt-4o-mini-tts", input: text, voice: "nova", // voz natural en espanol response_format: "opus", // formato nativo de Telegram instructions: "Habla en espanol con tono profesional y amigable." }); // POST a https://api.openai.com/v1/audio/speech // Headers: Authorization: Bearer {OPENAI_API_KEY} // Content-Type: application/json // Response: binary audio (Opus) return audioBuffer; // enviar a Telegram via sendVoice }

5.7 Consideraciones de UX para respuesta por voz

Cuanado responder con voz
Recomendacion: responder con voz solo cuando el usuario envio un voice message. Si envio texto, responder con texto. Esto respeta la preferencia del canal elegido por el usuario.

Alternativa: agregar un comando /voice on / /voice off para que el usuario elija.
Limites para TTS
• Maximo 2,000 tokens por request (aprox. 1,500 palabras en espanol)
• Respuestas muy largas deben truncarse o enviarse como texto + resumen en voz
• Latencia: 1-3s para respuestas cortas, aceptable para Telegram

6. Evaluacion: Realtime API de ChatGPT NUEVO en v2

6.1 Que es la Realtime API

La Realtime API de OpenAI permite conversaciones de voz bidireccionales en tiempo real via WebSocket. Fue lanzada en produccion como gpt-realtime en octubre 2025.

6.2 Precios

ModeloAudio inputAudio outputCosto aprox./min
gpt-realtime $32/1M tokens $64/1M tokens ~$0.30/min
gpt-4o-mini-realtime-preview $10/1M tokens $20/1M tokens ~$0.10/min

6.3 Evaluacion para nuestro caso de uso

NO recomendada para Telegram Commander La Realtime API esta disenada para conversaciones de voz continuas con baja latencia (tipo llamada telefonica), no para mensajes discretos de Telegram.

Razones para descartarla:
• 50-100x mas cara que el pipeline Transcribe + TTS ($0.30/min vs $0.006/min)
• Requiere WebSocket persistente — overhead innecesario para mensajes esporadicos
• El modelo no es Claude — la respuesta la genera GPT-4o, no Claude. Se pierde el contexto del proyecto
• No se integra con el flujo existente de executeClaude()
• Beneficio principal (baja latencia) irrelevante para Telegram (los mensajes ya son asincronos)
Arquitectura correcta: pipeline separado Usar Transcribe (voz → texto) + Claude (texto → respuesta) + TTS (respuesta → voz) es la arquitectura optima porque:
• Claude procesa la logica y el contexto del proyecto
• OpenAI solo maneja la conversion de/hacia audio
• Costo 50-100x menor
• Cada componente es reemplazable independientemente

11. Riesgos y Mitigaciones

Costo por uso de APIs
Anthropic API (imagenes): ~$0.004/imagen de 1000x1000px.
OpenAI Transcribe: $0.003-$0.006/minuto (segun modelo elegido).
OpenAI TTS: ~$0.01-$0.02/respuesta corta.
Mitigacion: Limitar tamanio de imagenes/audio, rate-limiting por chat, mostrar aviso de costo. Usar gpt-4o-mini-transcribe (50% mas barato) por defecto.
Timeout en transcripcion/TTS
Audios largos pueden tardar mas en transcribir; respuestas largas tardan mas en sintetizar.
Mitigacion: Limitar a 2 minutos de audio, timeout de 30s, truncar TTS a 2000 tokens, enviar "Transcribiendo..." / "Generando audio..." al chat con sendChatAction.
Seguridad de API keys
Las keys no deben quedar en el codigo fuente.
Mitigacion: Usar telegram-config.json (ya en .gitignore) o variables de entorno del sistema. Una sola OPENAI_API_KEY sirve para transcripcion Y TTS.
Vendor lock-in con OpenAI NUEVO
Usar OpenAI para transcripcion Y TTS crea dependencia en un solo proveedor.
Mitigacion: La arquitectura de pipeline permite reemplazar cualquier componente: si OpenAI sube precios o degrada servicio, se puede migrar transcripcion a Deepgram o TTS a otro proveedor sin cambiar el flujo general. Las funciones transcribeAudio() y textToSpeech() encapsulan la dependencia.

12. Recomendaciones ACTUALIZADO

1. Implementar imagenes primero
Es la feature mas directa y de mayor valor inmediato. Permite enviar capturas de pantalla, diagramas, errores visuales y que Claude los interprete en contexto. Solo requiere ANTHROPIC_API_KEY que probablemente ya este disponible.
2. Audio (transcripcion) como segunda fase
Usar gpt-4o-mini-transcribe como modelo por defecto: 50% mas barato que Whisper, 89% menos alucinaciones, excelente calidad en espanol. Opcionalmente exponer un comando /quality high para usar gpt-4o-transcribe cuando se necesite maxima precision.
3. Respuesta por voz (TTS) como tercera fase NUEVO
Usar gpt-4o-mini-tts con voz nova y formato opus. Activar automaticamente cuando el usuario envie un voice message (espejo del canal). La misma OPENAI_API_KEY sirve, no hay dependencia adicional. Agregar instrucciones de estilo: "Habla en espanol con tono profesional y amigable".
4. UX: Indicadores de progreso
Enviar mensajes intermedios al chat: "Analizando imagen..." / "Transcribiendo audio..." / "Generando respuesta de voz...". Telegram tiene sendChatAction(typing) y sendChatAction(record_voice) que muestran indicadores nativos automaticamente.
5. NO usar la Realtime API de ChatGPT
La Realtime API ($0.30/min) es 50-100x mas cara y esta disenada para conversaciones continuas, no para mensajes discretos de Telegram. Ademas, reemplazaria a Claude como motor de respuesta, perdiendo todo el contexto del proyecto. El pipeline Transcribe + Claude + TTS es superior en todos los aspectos para este caso de uso.

Alcance

Este reporte cubre la validación PO+QA del flujo completo de catálogo de productos para el rol Cliente: pantalla de inicio con lista de productos y acceso al catálogo (ClientHomeScreen), pantalla de catálogo con búsqueda y filtros por categoría (ClientCatalogScreen), pantalla de detalle de producto con selector de cantidad y carrito (ClientProductDetailScreen), y la store de carrito compartida (ClientCartStore).


Los tests E2E Android (Maestro) no pudieron ejecutarse automáticamente debido a problemas de conectividad TCP con el forwarder de Maestro en el entorno de CI. Se capturó evidencia visual mediante ADB. Los tests unitarios (kotlin-test + coroutines-test) se ejecutaron completamente con 38/38 PASS.

8. Gaps y deuda técnica
⚠️
GAP-1: Stock siempre muestra "No disponible"
ClientProductDetail no tiene campo stockQuantity. ProductDTO tampoco lo expone. La UI muestra siempre el fallback stockUnavailable. Se recomienda agregar stock al DTO o mostrar "Consultar disponibilidad".
⚠️
GAP-2: Emoji de producto hardcodeado (🛍️)
En ClientCatalogScreen el emoji de cada producto es siempre "🛍️" (hardcoded). No se mapea desde la categoría ni desde el producto. Afecta la diferenciación visual en catálogos con muchos productos.
⚠️
GAP-3: Maestro E2E no puede ejecutarse en CI actual
java.util.concurrent.TimeoutException en TcpForwarder (dadb). Requiere actualizar dadb o cambiar la estrategia de conexión al emulador. Workaround: usar ADB directo o Maestro Cloud.
ℹ️
GAP-4: ClientHomeScreen sin @BeforeTest en tests
ClientCatalogViewModelTest no llama setUp() en @BeforeTest anotado — usa un método setUp() sin anotación que se llama manualmente en el test addToCart. Menor inconsistencia que podría causar state leaks entre tests.
ℹ️
GAP-5: Sin tests de integración para resolveBusinessId()
ClientProductDetailViewModel.resolveBusinessId() lee de SessionStore o BuildKonfig. No hay test que cubra el caso donde ambos están vacíos y se lanza IllegalStateException.

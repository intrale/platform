QA Android del issue 1957: Chatbot para WhatsApp Business del negocio. Voy a verificar los ocho criterios de aceptacion. Arranco el build ya instalado, abro la app Intrale Negocios, ingreso con el usuario QA y selecciono el negocio QA Automation. El panel principal carga pero el listado de negocios devuelve error en metricas, lo cual no bloquea la verificacion del feature. Una vez dentro del panel, busco la entrada de menu que deberia llevarme al dashboard de Conversaciones de WhatsApp. No hay ninguna. Ni card en el dashboard, ni item en el drawer, ni boton en ninguna pantalla del flavor business. La clase ChatbotConversationsScreen existe en el codigo y se registro en el DIManager con la ruta slash business slash chatbot, pero nadie llama navigate hacia esa ruta desde la interfaz, por lo tanto el usuario final jamas puede alcanzar la pantalla. Resultado por criterio: el primero, el bot responde automaticamente en WhatsApp Business, no esta implementado porque no hay webhook ni integracion con WhatsApp Cloud API. El segundo, el bot conoce catalogo, precios, horarios y zonas de delivery, no esta implementado porque no hay bot. El tercero, responde preguntas frecuentes, no esta implementado. El cuarto, genera link directo a la app para pedir, no esta implementado. El quinto, escala al dueno cuando no sabe responder, no esta implementado. El sexto, las conversaciones se ven en el dashboard de la app de negocio, esta parcialmente implementado como scaffolding pero es inaccesible para el usuario y solo muestra el estado Proximamente. El septimo, el dueno puede tomar el control de una conversacion, no esta implementado. El octavo, las respuestas se marcan como bot cuando son automaticas, no esta implementado. Conclusion del QA: el issue requiere entrega completa frontend mas backend y el estado actual entrega solo scaffolding Android sin punto de entrada. Se rechaza el issue y se devuelve al dev para agregar al menos una entrada de navegacion visible hacia la pantalla Proximamente, y al backend dev para levantar el webhook, el bot con Claude API y el dashboard de conversaciones reales.

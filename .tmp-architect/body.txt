## Objetivo
Línea de tiempo agrupada por día con filtros (skill, resultado, issue) y búsqueda; cada entrada expandible con detalle (fases, rebotes, costo, links a log/PR/entregable); agregados del período visible. El historial se vuelve el archivo navegable de la solución de cada issue (conecta con EP-3).

## Criterios de aceptación
- [ ] Filtros + búsqueda + paginación por fecha
- [ ] Entregables parciales adjuntos por ejecución
- [ ] Header con ejecuciones, % aprobado y duración mediana del período
- [ ] Timestamps humanos ("hace 5 min")

### Patrón técnico recomendado

- **Lógica en `lib/`, no en el monolito.** `dashboard.js` ya tiene ~12.1k líneas (#4143). El cómputo de filtros/agrupación/paginación/agregados vive en `dashboard-slices.js` (función pura testeable); `dashboard.js` solo arma el array fuente, registra la ruta y delega. La vista solo renderiza.
- **Contrato del slice (sugerido):**
  ```js
  // historialTimelineSlice(state, ctx, { skill, resultado, issue, q, limit, cursor })
  //   -> { groups: [{ dayLabel, dayKey, items: [...] }],
  //        aggregates: { count, pctApproved, medianMs },
  //        nextCursor, total }
  ```
  Mediana con `_percentile(durations, 50)` (reusar `:959`). `pctApproved = approved / count`.
- **Búsqueda literal, NO regex cruda (REQ-SEC-5 / CA-6).** Match `String.prototype.includes` case-insensitive sobre `titulo` / `skill` / `motivo_rechazo`. Si se usa `RegExp`, escapar metacaracteres primero. Límite máximo de resultados por request obligatorio (paginación server-side).
- **Timestamps humanos (CA-5).** Helper SSR de tiempo relativo ("hace 5 min", "hace 2 h") manteniendo el timestamp absoluto en `title=` (precisión). `renderHistCard` ya formatea con `toLocaleString` (`:78-80`) — extender, no romper.
- **Links a entregables/logs (REQ-SEC-2).** Reusar el saneo existente `path.basename(x).replace(/[^a-zA-Z0-9\-\.]/g,'')` + confinamiento a base dir (`/logs/view/` en `:10341`, `/logs/` en `:10352`) y la whitelist `isSafeFilename` (`historial.js:58`). Prohibido construir paths con input crudo, `..` o rutas absolutas.
- **Entregables parciales (CA-2).** Fuente: `lib/skill-deliverable-attachments.js` (`getSkillSourcesCatalog`, recolección issue-scoped `[{type,path,descriptor}]`) + `lib/write-deliverable.js`. Conexión **soft** con EP-3 #3926 (OPEN): la vista degrada elegante si un entregable no existe (CA-3), no se bloquea.

### Riesgos identificados

- Riesgo: **engrosar el monolito `dashboard.js`** (#4143). | Mitigación: toda la lógica nueva va en `dashboard-slices.js`; el monolito solo arma el array, registra ruta y delega render.
- Riesgo: **ReDoS / DoS** por query del usuario o historial grande (REQ-SEC-5). | Mitigación: match literal `includes` (sin `new RegExp` crudo) + límite máximo por request + paginación server-side.
- Riesgo: **XSS reflejado** del término de búsqueda y datos dinámicos (REQ-SEC-1). | Mitigación: todo dato dinámico (título, skill, `motivo_rechazo`, chips de filtro, **echo del query**) pasa por `escapeHtmlText`/`escapeHtmlAttr` (`lib/escape-html.js`, ya importado en `historial.js:48`).
- Riesgo: **granularidad de costo por ejecución** — el costo hoy se agrega por issue/skill/fase, no por ejecución individual. | Mitigación: best-effort join por (issue,skill,fase,ts); si no hay match exacto, degradar a "costo a nivel issue/fase" (CA-3 lo permite).
- Riesgo: **retención/profundidad del historial** — `matrixEntries` solo cubre issues con markers en FS; `metrics-history.jsonl` es muestreo de recursos, no log de ejecuciones. | Mitigación: paginar sobre lo disponible (markers `procesado/` + entregables archivados); persistencia de largo plazo fuera de scope (anotado por guru, no bloquea).
- Riesgo: **path traversal / secrets en entregables servidos** (REQ-SEC-2/3). | Mitigación: saneo de filename + confinamiento a base dir; contenido de texto por `redactLogText` (`:10362`); PDFs binarios — si se adjuntan entregables PDF, validar que no expongan secrets.
- Riesgo: **regresión del dashboard** al reemplazar la lista plana. | Mitigación: mantener el contrato `renderHistorialSsr(state, opts)` + fallback a string vacío (`dashboard.js:3746-3751`); suite `historial.test.js` verde antes de cerrar.
